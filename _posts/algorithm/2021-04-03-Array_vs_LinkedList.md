배열과 리스트의 시간 복잡도를 비교하여 어떠한 상황에서 어떤 자료구조가 더 효율적으로 사용될 수 있는지 정리하기 위해 작성했으며, 두 가지 자료구조의 기본적인 개념과 동작 원리를 알고있다는 가정 하에 간단하게 작성하였습니다.

## 데이터 접근 속도

`Array` 는 인덱스를 사용하여 원소에 빠르게 접근할 수 있습니다.

**시간복잡도 O(1)**

`Linked List` 는 특정 원소에 접근하기 위해 순차적 접근 방식을 사용합니다.

**시간복잡도 O(n)**

## 데이터 삽입 속도

→ 경우에 따라 다르지만, 보통 다음과 같습니다.

`Array` 배열의 맨 끝에 삽입 할 경우 O(1) 시간복잡도로 가능하지만, 배열의 중간에 삽입하는 경우 데이터를 한칸 씩 미루는 비효율적인 과정이 필요합니다.

**시간복잡도 O(n)**

`Linked List` 는 삽입할 위치를 찾는 과정에서 O(n) 시간복잡도가 소요되지만, 노드에 대한 포인터 정보만 수정하여 삽입이 가능합니다. 따라서 Array보다 성능이 좋습니다.

**시간복잡도 O(n)**

## 데이터 삭제 속도

→ 경우에 따라 다르지만 보통 다음과 같습니다.

`Array` 배열의 중간에 있는 원소를 삭제하는 경우 배열의 한칸 씩 바꿔주는 비효율적인 과정이 필요합니다.

**시간복잡도 O(n)**

`Linked List` 는 삽입할 위치를 찾는 과정에서 O(n) 시간복잡도가 소요되지만, 노드에 대한 포인터 정보만 수정하여 삭제가 가능합니다. 따라서 Array보다 성능이 좋습니다.

**시간복잡도 O(n)**

## 메모리 할당 방식

`Array` 의 경우, 배열이 선언 될 때, Stack 영역에 정적 메모리 할당이 이루어집니다. (Compile Time)

`Linked List` 의 경우, 새로운 노드가 추가될때, Heap 영역에 동적 메모리 할당이 이루어집니다. (Run Time)

# 결론

- 삽입과 삭제가 빈번한 경우 `Linked List` 방식을 사용하는 것이 좋습니다.
- 데이터에 접근하는 것이 중요한 경우 `Array` 방식을 사용하는 것이 좋습니다.
- 이때, 사용할 메모리의 범위를 지정해 주는 경우, `Array` 방식이 더 유리할 수 있습니다. (List가 변화될 때 마다 메모리의 할당과 해제가 이루어지는 `시스템 콜` 을 사용하는 구문은 시간이 시간복잡도에 포함되지 않지만, 시간이 꽤 소요됩니다. )
